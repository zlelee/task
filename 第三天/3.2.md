四 简答题

1 .call()和.apply()的含义和区别?

call 和 apply 都是改变 this 指向的,区别在于它们接收的参数方式不同, call 第一个参数是需要 this 指向的对象,后面是可以接收一个个的参数,apply 后面接收的第一个参数和 call 相同,但是第二个参数必须接受一个数组

2 谈谈对原型和闭包的理解？

在 js 中每个对象都有一个 prototype 属性,指向了另外一个对象,这个对象的所有属性和方法,都会被构造函数的实例继承,这样我们就可以把公用的属性和方法定义在 prototype 对象属性上

3 简单介绍一下 promise,他解决了什么问题？
解决了回调地狱,当我们做一些异步操作的时候,就会出现嵌套的情况,当嵌套的越来越多就会让我们代码变得臃肿,而用 promise 的 then 就可以避免嵌套

4 普通函数和箭头函数 this 指向说明？

普通函数的 this 指向 window,箭头函数中的 this 指向上下文中的 this

5 let var const 的区别是什么？
var 有变量提升 没有块级作用域
let 没有变量提升 有块级作用域 定义的是变量
const 没有变量提升 有块级作用域 定义的是常量

6 说一下 es6 的新特性和 es6 新增的数组方法不少于 6 个？

es6 新增了变量声明 let 和 const, 新增了箭头函数简化了函数的写法,并且可以巧用箭头函数中没有 this 这一特点, 新增了对象解构,新增了剩余参数,新增了 class 类的概念代替构造函数,新增了 import 导入导出
es6 新增的数组方法 filter forEach some reduce map every
7 ==和===有什么区别？JavaScript 基本数据类型有哪些？

== 是相同 === 是严格相同 ,当 == 来比较时,会比较两边的值是否相同,如果不相同就会进行数据类型转换,再进行比较,而===是比较两边的值是否完全相等

8 什么是事件委托？请写出一个事件委托？

事件委托也称为事件代理,事件委托是把原本需要绑定在子元素的响应事件委托给父元素让父元素担当监听的职务,事件委托的原理是 dom 元素的事件冒泡

9 null 和 undefined 的区别？

null 表示"没有对象"，即该处不应该有值
undefined 表示"缺少值"，就是此处应该有一个值，但是还没有定义

10 HTML5 有哪些新特性？

html5 新增了很多语义化标签, 还有音频 audio 和视频 video 标签
还有新增了表单的属性和方法,还有 canvas 绘图,以及 web storage

11 跨越问题如何解决？

1.jsonp,通过 script 的 src 属性没有跨域限制,通过回调函数的参数来接受数据,不过只能够使用 get 请求
2.cors,通过在请求头中添加需要的字段来实现跨域请求

12 ajax 工作流程以及原理？

1. 通过 xmlHttpRequest 构造函数示例一个 xhr 对象
2. 调用 open()方法定义请求的方式,请求的地址以及请求是否异步的布尔值
3. 调用 send()方法向服务器发送请求
   send()方法接收一个参数,是作为请求体发送的具体数据,如果不需要请求体则需要传递一个 null
4. 监听 readystatechange 事件,监听 ready state 属性的值,当 readystate 为 4 时,表示数据已经准备就绪,可以使用了
